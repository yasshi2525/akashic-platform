# CHANGELOG

## 1.4.0

- Feature
  - ゲームログ閲覧ページを追加 (`/game/{id}/logs`)
    - ゲーム詳細ページに投稿主のみ表示される「ログを見る」ボタンを追加
    - 終了したプレイを一覧表示。部屋名・部屋主（アイコン付き）・開始/終了時刻を表示
    - クラッシュ・エラーあり・エラーなしをバッジでひと目確認できる
    - エラーの詳細メッセージはクリックで展開してオンデマンド取得
    - ログのダウンロードリンクはアップロード完了後に表示
  - ログアップロードをノンブロッキング (fire-and-forget) に変更
    - プレイ終了時にゲームの停止・DB 更新を完了した後、アップロードをバックグラウンドで実行
    - S3 Multipart Upload に対応 (`@aws-sdk/lib-storage`)。5MB 単位でパートを送信し、メモリの使用量を抑制
    - アップロード完了後に DB の `logUploadedAt`・`crashed`・`errorLogged` を更新し、通知を作成
  - 通知の送信タイミングをログアップロード完了後に変更
    - プレイ中のエラー・クラッシュ発生時は内部フラグを立てるのみとし、データが揃ってから通知する
    - 通知リンクを `/game/{gameId}/logs#play-{playId}` に変更
  - Play レコードを終了後も保持するように変更
    - 終了時に削除する代わりに `isActive = false`・`endedAt` を設定

## 1.3.1

- Feature
  - ゲームの異常終了・エラーログ出力をゲーム投稿主に通知する機能を追加
    - ゲームが異常終了した場合 (`errorTrigger` で catch)、投稿主に **「ゲームがエラーで異常終了しました」** と通知する (重要度: 高)
    - プレイ中に `console.error` への出力があった場合、投稿主に **「ゲームの実行中にエラーログが出力されました」** と通知する (重要度: 低、同一プレイで1回のみ)
    - いずれの通知からも、該当プレイのログビューアー (`/game-log/{contentId}/{playId}`) へ直接遷移できる
  - プレイのゲームログを S3 へアップロードする機能を追加
    - プレイ終了時に、そのプレイで出力された全ログ (info / warn / error) を JSONL 形式で S3 に保存する (`play-logs/{contentId}/{playId}.jsonl`)
    - ログは投稿主のみ閲覧可能 (`GET /api/play-log/{contentId}/{playId}`)
